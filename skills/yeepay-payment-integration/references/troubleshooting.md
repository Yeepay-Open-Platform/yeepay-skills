# 排障与上线（汇总）

本文件汇总各业务域排障、回调问题、错误码定位、对账差异处理与上线检查。**回调验签/上线规则为本地权威，不走 curl**；接口业务错误码走 curl。

## 一、错误码两层定位

接口返回错误时按顺序排查：

1. **业务级错误码（优先）**：当前调用接口自身的错误码。
   - 在 `产品能力/api-index.yaml` 找到该接口的 `doc_md` 链接。
   - `curl -sS "<doc_md>"`，查看「错误码」章节，按说明处理。
   - 形如 `00012 产品未开通`、`00001 参数错误`，含义因接口而异。
2. **平台级错误码（再查）**：鉴权/签名/IP/限流等平台问题。
   - 本地：`平台文档/开始对接/平台错误码说明.md`。
   - 形如 `40xxx`、`sys.xxx`、`isv.xxx`。
3. **仍无法判断**：走下方通用排障（签名/鉴权/网络/参数）。

## 二、回调问题排查

常见：回调收不到 / 验签失败 / 重复通知 / 与查单状态不一致。

规范来源（本地）：
- `平台文档/平台规范/安全认证/结果通知(RSA).md`
- `平台文档/平台规范/安全认证/结果通知(SM2).md`
- `平台文档/平台规范/结果通知机制说明.md`

排查顺序：
1. 回调地址与网络可达性：`notifyUrl` 是否公网可达、是否被网关/防火墙拦截；商户防火墙是否已放行易宝出口（见 `平台文档/平台规范/回调网络配置.md`）。
2. 验签规则与密钥配置：`平台文档/平台规范/安全认证/鉴权认证机制(RSA).md`；公钥/证书与环境是否匹配。
3. 回调报文是否与验签输入一致（字段、编码、顺序）。
4. 回调处理幂等：重复通知不得重复入账。
5. 控制台 **结果通知** 页可查询/重发通知（见 `平台文档/平台规范/结果通知查询与重发.md`）。
6. 最终状态以查单兜底确认（回调与查单不一致时以查单为准）。

## 三、签名/验签失败

- 核对密钥配对关系（商户私钥 / 平台公钥）。
- 核对签名串拼装、参数排序与 UTF-8 编码、URL 安全 Base64。
- 核对 `x-yop-content-sha256` 计算规则（GET 对空串、POST+JSON 对完整体、form 对非文件参数排序编码；**form 无参数时须对空串 `""` 计算摘要**）。
- Java SDK 用户：查 `平台文档/开始对接/Java-SDK报错说明.md`（含 env=qa、软算法包、BC 版本、HTTP 204 等案例）。
- 详见 `平台文档/平台规范/安全认证/`。

## 四、收单排障

- 同步返回"支付中"不是失败（被扫/风控场景）：轮询查单确认，勿重复下单。
- `userId`/`openId` 报错：确认 `openId` 基于支付用 `appId` 获取。
- `prePayTn` 唤起失败：按 `产品能力/收单/prePayTn唤起方式速查.md` 确认 `payWay + channel + scene` 对应的返回类型与唤起方式（catalog `prepay-tn-usage`）。
- 终态以「通知 + 查单」为准。

## 五、退款/出款排障

- `account`（账户）、`balance`（代付代发）分组接口（如提现 `/rest/v1.0/account/withdraw/order`、代付 `/rest/v1.0/balance/transfer_send`）调用被拒或报签名/权限类错误：先确认应用密钥是否为 **CFCA 证书**——该两个分组的资金出款类接口必须使用 CFCA 证书签名，普通 RSA 文本密钥会被平台拒绝；同时确认 IP 白名单已配置。见 `平台文档/接入准备/密钥管理/CFCA证书介绍.md`、`平台文档/开始对接/配置IP白名单.md`。
- 状态长期不变：先查询接口核对状态机，再查回调是否到达。
- 写操作（退款/结算/提现）单号业务唯一；重试用同一单号，**严禁重复出款**。
- 超时/未知结果：先查后重试。

## 六、对账差异处理

- 以易宝账单为基准与商户流水勾兑；差异分类：己方多 / 己方少 / 对方多 / 对方少 / 状态不一致。
- 单笔差异回到对应业务域用查询接口核实终态后再调整。
- 注意账单生成时延、`yos` 与 `openapi` 网关区分、金额单位与符号。

## 七、上线检查清单（关键项，未通过不建议上线）

- [ ] 测试与生产参数完全隔离。
- [ ] 请求签名、回调验签联调通过。
- [ ] 回调处理具备幂等能力（重复通知不重复入账）。
- [ ] 支付/退款/分账/出款均可通过查询接口确认终态。
- [ ] 关键日志已脱敏，不输出密钥与完整敏感标识（卡号/证件号）。
- [ ] 异常重试策略明确（先查状态再重试）。
- [ ] IP 白名单/回调网络已配置（`平台文档/开始对接/配置IP白名单.md`、`平台文档/平台规范/回调网络配置.md`）。
- [ ] 回滚预案与责任人明确。
- [ ] `api-index.yaml` 中相关接口的 `doc_md` 已 `curl` 可达。

> 详尽上线规范见 `平台文档/平台规范/上线前检查清单.md`。
