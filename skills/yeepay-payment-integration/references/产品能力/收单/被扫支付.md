# 被扫支付（付款码）

**商户扫用户**的付款码完成收款（用户出示微信/支付宝/云闪付付款码，商户扫码枪/收银台扫描）。注意：如果使用易宝采购或绑定过的pos机设备，无需进行接口对接，仅在老板收钱app内完成配置即可使用，该场景适用于商户使用自己的扫码设备。

> 接口字段以在线文档为准：按下表 catalog id 在 `../api-index.yaml` 取其 `doc_md`，执行
> `curl -sS "<doc_md>"` 后再实现（单文件含字段/示例/错误码/示例代码）。

## 场景 → 接口


| 用途    | catalog id           | 方法   | 路径                             |
| ----- | -------------------- | ---- | ------------------------------ |
| 付款码支付 | `aggpay-barcode-pay` | POST | `/rest/v1.0/aggpay/pay`        |
| 查单    | `trade-order-query`  | GET  | `/rest/v1.0/trade/order/query` |


支付结果回调：`aggpay-barcode-pay` 的 `notify_spi: trade.pay-result`。

## 前置条件

- 收单商户开通：商家扫码支付宝线下、商家扫码微信线下两个产品。
- 商户完成微信、支付宝子商户号认证。

## 下单关键参数（`/rest/v1.0/aggpay/pay`）


| 参数                | 说明                                     |
| ----------------- | -------------------------------------- |
| `merchantNo`      | 收款商编（必传）                               |
| `orderId`         | 请求订单号，商户自定义（必传）                        |
| `orderAmount`     | 订单金额（必传）                               |
| `payWay`          | `MERCHANT_SCAN`（必传）                    |
| `authCode`        | 付款码；刷脸付时先取 authInfo→facecode 作为授权码（必传） |
| `userIp`          | 用户真实支付 IP（必传）                          |
| `terminalId`      | 扫码设备编号（必传）                             |
| `scene`           | `OFFLINE`（必传）                          |
| `channel`         | 渠道类型（选传，可由付款码自动识别）                     |
| `expiredTime`     | 订单截止时间，不传默认 24h                        |
| `notifyUrl`       | 支付结果通知地址                               |
| `fundProcessType` | 分账标识（按需）                               |


## 同步返回与终态

- 同步返回 `status` 可能为 `SUCCESS`，也可能为**处理中**（需用户输密码/风控校验）。
- 处理中场景必须**轮询查单**确认终态，**不可**立即判失败或重复下单。
- 异步通知到 `notifyUrl`；查单 `trade-order-query` 必传 `merchantNo` + `orderId`。

## 易错点

- `payWay=MERCHANT_SCAN`（被扫专用），与主扫 `USER_SCAN` 不同。
- 付款码 `authCode` 有时效，扫码后尽快提交。
- `terminalId`（设备编号）、`userIp` 为必传。
- 与主扫区别：被扫=商户扫用户；主扫=用户扫商户。
