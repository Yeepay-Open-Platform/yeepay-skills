# 主扫支付（微信/支付宝聚合码）

**用户扫商户**展示的**一码多渠道聚合码**完成支付（同一张码，微信/支付宝都能扫）。

> 接口字段以在线文档为准：按下表 catalog id 在 `../api-index.yaml` 取其 `doc_md`，执行
> `curl -sS "<doc_md>"` 后再实现（单文件含字段/示例/错误码/示例代码）。

## 场景 → 接口


| 用途                   | catalog id                 | 方法   | 路径                                    |
| -------------------- | -------------------------- | ---- | ------------------------------------- |
| 生成聚合订单码              | `aggpay-pay-link`          | POST | `/rest/v1.0/aggpay/pay-link`          |
| 查单                   | `trade-order-query`        | GET  | `/rest/v1.0/trade/order/query`        |
| 公众号 appid+授权目录配置（必读） | `aggpay-wechat-config-add` | POST | `/rest/v2.0/aggpay/wechat-config/add` |


支付结果回调：`aggpay-pay-link` 的 `notify_spi: trade.pay-result`。

## 业务说明

- 生成聚合码，用户用微信或支付宝扫码均可拉起支付。
- 微信通过公众号包装支付，支付宝通过生活号包装支付。

## 前置条件

- 收单商户开通：用户扫码支付宝线上、用户扫码微信线上两个产品。
- 商户完成微信、支付宝子商户号认证。
- 完成微信公众号 appid、appSecret、支付授权目录配置。

## 接入步骤

1. **公众号配置**：调 `aggpay-wechat-config-add`，`appIdList`（`appIdType=OFFICIAL_ACCOUNT`）+ `tradeAuthDirList`（授权目录）+ 必要时 `reportMerchantNo`；返回 `configResult` 中各项 `status=ACCEPT` 为成功。
2. **生成聚合码**：调 `aggpay-pay-link`，关键参数：
  - `merchantNo`/`orderId`/`orderAmount`/`goodsName` 必传
  - `payWay=USER_SCAN`
  - `scene` 按渠道传，如 `{"WECHAT":"OFFLINE","ALIPAY":"OFFLINE"}`
  - `userIp`、`notifyUrl`、`fundProcessType`（按需）
  - 返回 `qrCodeUrl`（如 `https://qr.yeepay.com/...`）
3. **展示二维码**：用 `qrCodeUrl` 生成二维码展示，用户扫码进入聚合收银台支付。
4. **通知 + 查单**确认终态。

## 易错点

- 聚合码用 `aggpay-pay-link`（返回 `qrCodeUrl`），不是 `aggpay-pre-pay`。
- `scene` 为按渠道的 JSON 串，注意格式。
- 需每渠道独立码用 `主扫支付（独立码-线上PC）.md`。
- 终态以后端为准。
