# 余额分账

基于商户**账户余额**进行分账（不依赖具体某笔原交易）。

> 接口字段以在线文档为准：按下表 catalog id 在 `../api-index.yaml` 取其 `doc_md`，执行
> `curl -sS "<doc_md>"` 后再实现（单文件含字段/示例/错误码/示例代码）。

| 用途 | catalog id | 方法 | 路径 |

## 申请入账方

商户可通过此接口，为子商户**申请添加入账方**。添加入账方后，商户可使用易分账能力，将资金分账至入账方的银行账户。

适用场景：开通「余额分账_至入账方_对公」或「余额分账_至入账方_对私」产品后，分账接收方不是已入网商户、而是需分账至其银行账户的入账方时，须先完成入账方申请。

接入步骤：

1. 调 `mer-receiver-apply` 申请添加入账方（对私入账方通过 `notify_spi: mer.receiver-apply-receiver-notify` 接收结果通知；对公同步返回，无回调）。
2. 申请提交后，调 `mer-receiver-apply-progress-query`，传入申请时的 `requestNo` 查询进度与审核结果。
3. 入账方添加成功并完成签约（对公需双向签约、对私需入账方签约及商户确认，见上表产品备注）后，再发起余额分账；分账明细中 `ledgerType=RECEIVER`，`ledgerNo` 传入账方编号。

## 接入要点

1. **接收方准备**：
   - 分账给其他商户：接收方需入网（特约商户入网企业/个体/小微）。
   - 分账给入账方：先完成上文「申请入账方」流程，把接收方添加为该商户的入账方（`ledgerType=RECEIVER`）。
2. **发起分账**：调 `divide-balance-apply`。
   - 一个批次最多 30 笔明细，按笔分别处理、按笔分别回调。
   - `parentMerchantNo` 与 `merchantNo` 需符合易宝商户关系（同级或上下级）。
   - `ledgerType=MERCHANT/MERCHANT_BINDCARD/BANKACCOUNT` 时 `ledgerNo` 传易宝商编，且需与 `merchantNo` 同属一个 SAAS/平台。
3. **金额与手续费**：`divideAmount`/`feeAmount`/`feeChargeSide`(INSIDE/OUTSIDE) 决定 `debitAmount`/`receiveAmount`，规则以在线文档为准。
4. **确认结果**：响应 `code=SR000000` 仅表示受理，最终以 `status`（查询接口/通知）为准。

## 易错点

- 余额分账基于账户余额；订单分账基于原交易（见 `订单分账.md`），两条链路不要混淆。
- 接收方未入网/未添加为入账方会失败。
- `code=SR000000` ≠ 分账成功，必须查 `status`。
- 分账单号业务唯一，重试用同一单号。
