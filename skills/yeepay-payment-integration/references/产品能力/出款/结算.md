# 结算

将易宝账户资金结算到银行卡/资金账户（自助结算）。

> 接口字段以在线文档为准：按下表 catalog id 在 `../api-index.yaml` 取其 `doc_md`，执行
> `curl -sS "<doc_md>"` 后再实现（单文件含字段/示例/错误码/示例代码）。

> 结算接口属 `settle` 分组（`/rest/v1.0/settle/...`），**不在 CFCA 证书强制范围内**（强制范围为 `account`/`balance` 分组，见
> `../../平台文档/接入准备/密钥管理/CFCA证书介绍.md`）；如同时使用提现请注意其证书前置条件（`提现.md`）。

## 场景 → 接口

| 用途 | catalog id | 方法 | 路径 |
|------|-----------|------|------|
| 自助结算申请 | `settle-self-apply` | POST | `/rest/v1.0/settle/self-settle/apply` |
| 结算记录查询 | `settle-records-query` | GET | `/rest/v1.0/settle/records/query` |

结算结果回调：「自助结算结果通知」（下单传 `notifyUrl` 才通知）。

## 业务流程图

```dot
digraph 自助结算流程 {
    // ── loop A：商户发起自助结算（可按结算周期重复）──

    易宝支付 -> 商户业务系统   [label="5. 返回商户结算结果"];

```

## 流程

1. 商户调 `settle-self-apply` 申请自助结算（传 `notifyUrl` 以接收结果）。
2. 易宝生成结算订单，异步通知结算结果。
3. 也可主动调 `settle-records-query` 查询结算结果。

> 自动结算 vs 自助结算由**产品码**区分；结算方向由 `settlementDirection` 配置：`ACCOUNT`（结算到资金账户）/`BANKCARD`（结算到银行账户），开通结算产品时必填。

## 自助结算申请关键参数

| 参数 | 说明 |
|------|------|
| `merchantNo` | 收款商户编号（必填） |
| `settleRequestNo` | 商户结算请求号，商户端不重复（必填） |
| `operatePeriod` | 结算时间范围：`PERIOD`(指定区间)/`ALL`(全部可结算)/`AMOUNT`(指定金额，仅班次结算可用)（必填） |
| `endTime` | `operatePeriod=PERIOD` 时必填（D0 到秒，D1/T1 到日） |
| `settleAmount` | `operatePeriod=AMOUNT` 时必填 |
| `parentMerchantNo`/`notifyUrl`/`bankRemark`/`extInfo` | 非必填 |

> 建议商户 0:05 之后再发起自助结算。响应 `code=000000` 仅代表受理，实际结果以「结算结果通知」的 `settleStatus`（`SUCCESS`/`SETTLE_FAIL`）为准。

## 结算记录查询关键参数

- 必填：`merchantNo`。
- 条件必填：`settleRequestNo` 与（`settleRequestBeginTime`+`settleRequestEndTime`）二选一，不能同时为空。
- 返回含结算订单状态 `status`：`SUCCESS`/`BEFORE_REMIT`(待出款)/`SETTLE_FAIL`/`REMITTING`(银行处理中)，及到账明细。

## 结算结果通知

- 通知失败默认最多重试 3 次，每次间隔 300 秒。
- 关键字段：`settleStatus`、`settleAmount`、`feeAmount`、`retreatFlag`(是否冲退)、出款明细 `settleRecordDetailsDtos`。

## 易错点

- 结算（资金→银行卡/账户）与提现（账户余额提现）是不同链路，见 `提现.md`。
- `settleRequestNo` 业务唯一；**严禁重复出款**，超时先查再重试。
- `code=000000` ≠ 结算成功，以 `settleStatus` 为准。
