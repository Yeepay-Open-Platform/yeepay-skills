# 常见问题

## 网关返回 204 / 503？

- **204**：网关对非法请求短路处理，响应体为空。常见原因：`server_root` 与 `apiUri` 拼接错误（如 `yop-cente` 少字母）、签名/`x-yop-content-sha256` 错误、AppKey 或请求方法不符。Java SDK 详见 `开始对接/Java-SDK报错说明.md` §五。
- **503**：YOS 等服务可能限速、限并发，请按业务场景合理使用并配合重试与查单。

## 什么是数字签名？

摘要对数据做哈希，再用私钥对摘要非对称加密即得签名。验签方用公钥解密并比对摘要，确认来源与完整性。

## 什么是数字信封？

1. 用对称密钥加密明文得密文；
2. 用接收方公钥加密对称密钥得数字信封；
3. 发送信封 + 密文；
4. 接收方私钥解开信封得对称密钥，再解密密文。

RSA 回调四段密文格式见 `平台规范/安全认证/结果通知(RSA).md` §报文结构；非 SDK 实现见 `平台规范/安全认证/回调解密协议.md`。

## 应用证书涉及哪些角色？

通常由开放平台账户管理者上传商户公钥并记录 AppKey；开发人员获得 **AppKey、商户私钥、易宝公钥** 写入配置或代码完成签名与验签。

## 和开放平台交换公钥是什么意思？

商户私钥本地保管，公钥上传平台；平台提供易宝公钥给商户。商户请求用私钥签名、平台用商户公钥验签；平台通知用平台私钥签名、商户用易宝公钥验签。
