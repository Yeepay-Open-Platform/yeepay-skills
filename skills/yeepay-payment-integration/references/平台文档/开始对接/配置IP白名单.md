# 配置IP白名单

## 一、什么是IP白名单？

为提高商户服务器访问 API 网关的安全性，易宝开放平台提供 **应用级 IP 白名单**：网关校验**商户请求**的来源 IP，不在白名单内的请求将被拦截。

以下资金类敏感接口**必须**配置 IP 白名单：

- `/rest/v1.0/balance/yop-group-remit/transfer-confirm`
- `/rest/v1.0/balance/transfer_batch_send`
- `/rest/v1.0/balance/transfer_send`
- `/rest/v1.0/balance/cash`

> 易宝**回调商户**时的网络放行见 `平台规范/回调网络配置.md`（与本文方向相反）。

## 二、进入配置页

**路径**：登录 [控制台](https://open.yeepay.com/developer-center) → **我的应用** → **IP白名单**。

首次使用点击 **立即设置**；已有规则可在 **IP 白名单管理** 页新增、修改、删除、查询。

平台支持两种范围：

- **部分接口**：仅对已签约的高风险接口生效。
- **全部接口**：对该应用所有接口生效（不含返回 302 的重定向接口）。

若同一接口同时存在部分与全部规则，**以部分接口规则为准**，全部接口规则对该接口被忽略（见 §六）。

## 三、IP 格式

每行一个地址，支持：

1. 单 IPv4（如 `123.103.49.137`）
2. IPv4/掩码（如 `123.103.49.137/24`）
3. 单 IPv6（如 `2409:8800:8813:208a:617a:6125:d33f:79e0`）
4. IPv6/前缀（如 `2409:8800:8813::/64`）
5. 以上格式组合（多行）

配置的是**商户服务器出口 IP**，配置前请核对正确性与完整性。

## 四、新增规则

### 全部接口

1. **IP 白名单管理** → **添加规则** → 选择 **全部接口** → **下一步**。
2. 在文本框逐行填写出口 IP → **确认**。

### 部分接口

1. **添加规则** → 选择 **部分接口**（页面展示当前应用已签约的高风险接口列表；非高风险接口须选「全部接口」）。
2. 勾选要保护的接口 → **下一步**。
3. 填写 IP 列表 → **确定**。

## 五、修改与删除

- **修改**：在规则操作列点 **修改** → 安全验证 → 在 **修改规则** 中更新接口勾选或 IP 列表 → **确认**。
- **删除**：操作列点 **删除** → **确认删除** → 安全验证通过即删除。

## 六、拦截规则（场景 3 示例）

应用 A 存在四条规则：

| 规则 | 类型 | 接口 | IP |
| --- | --- | --- | --- |
| 1 | 部分 | `/rest/v1.0/std/trade/order` | 192.168.10.1 |
| 2 | 部分 | `/rest/v1.0/std/trade/order`、`/rest/v1.0/std/trade/orderclose` | 192.168.10.2 |
| 3 | 部分 | `/rest/v1.0/std/trade/orderquery` | 192.168.10.3 |
| 4 | 全部 | （所有接口） | 192.168.10.4 |

调用 `/rest/v1.0/std/trade/order` 且来源 IP 为 `192.168.10.4` 时：网关只按规则 1、2 校验，**不会**使用规则 4，请求将被拦截。
