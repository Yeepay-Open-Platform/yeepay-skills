# 结果通知机制说明

本文说明易宝异步结果通知的**通用机制**（地址、重试、应答、幂等、查单兜底）。报文加解密与验签细节按密钥类型见：

- RSA：`平台规范/安全认证/结果通知(RSA).md`
- SM2：`平台规范/安全认证/结果通知(SM2).md`

## 一、notifyUrl 要求

- 须为**公网可达**的 **HTTPS** 地址，在下单/申请类接口中传入（字段名多为 `notifyUrl`，以接口 doc_md 为准）。
- 商户侧须放行易宝出口访问（见 `平台规范/回调网络配置.md`）。
- 地址填错可在控制台 **结果通知** 页查询并重发（见 `结果通知查询与重发.md`）。

## 二、通知与查单双通道

- 交易/业务**终态**以 **回调 + 查单** 共同确认，不以前端页面或同步返回为准。
- 未收到通知或处理失败时，须实现**主动查单**补偿；回调与查单不一致时**以查单为准**。

## 三、幂等与重复通知

- 同一笔业务可能因重试**多次通知**，商户处理须**幂等**（按业务单号去重，避免重复入账/重复出款）。
- 写操作重试使用**同一业务单号**，超时/未知先查状态再决定是否重试。

## 四、平台重试与商户应答

商户处理成功时须返回约定应答，否则平台判定失败并**自动重试**（具体频次因产品/场景略有差异，以控制台发送记录为准）。

| 密钥类型 | 成功应答 |
| --- | --- |
| RSA | HTTP **200**，响应体纯文本 **`SUCCESS`**（无需加密签名） |
| SM2 | HTTP **200**，响应体 JSON **`{"result":"SUCCESS"}`**，响应头含 `x-yop-sign`（SDK 可自动签名） |

处理失败时返回 4xx/5xx 或不符合上表格式，平台将按策略重试。运维侧查询/手工重发见 `结果通知查询与重发.md`。
