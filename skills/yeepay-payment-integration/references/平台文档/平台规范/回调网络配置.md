# 回调网络配置

本文说明商户侧如何放行易宝异步通知（结果通知）的网络访问：包括防火墙白名单策略、出口 IP/网段清单及维护建议。

> 商户**请求**易宝 API 时的来源 IP 白名单，见 `开始对接/配置IP白名单.md`（应用级 IP 白名单）。本文聚焦**易宝回调商户**时的网络放行。

## 一、为什么需要配置

商户服务器需能接收易宝发起的 HTTPS 异步通知。若贵司在网络边界部署了防火墙，通常需要将易宝出口 IP 或网段加入白名单，否则可能出现「回调收不到」。

**重要**：易宝出口 IP 可能因 ISP 变更、扩容等原因调整，**硬编码固定 IP 列表存在失效风险**。建议优先使用域名白名单或定期 DNS 解析。

## 二、推荐配置方式（按优先级）

### 1. 域名白名单（推荐）

若网络设备支持域名白名单，请放行易宝出口域名：

- `out.yeepay.com`

### 2. 定期 DNS 解析

若只能使用 IP 白名单，建议**每小时**解析一次 `out.yeepay.com`，将解析结果动态同步到防火墙规则。

### 3. 参考 IP 清单（辅助）

平台不定期维护的 [IP 白名单列表](https://yeepay.feishu.cn/docx/F2q3dKHckokUvsxVnP9chPZTnRf) 可作参考，但更新间隔不确定，**不宜作为唯一依据**。

## 三、添加到网络防火墙

防火墙设备型号各异，请参考贵司设备说明书，将第二节中的域名或 IP/网段加入**入站**白名单，放行易宝到商户 `notifyUrl` 的 HTTPS 访问。

操作要点：

1. 确认 `notifyUrl` 为公网可达的 HTTPS 地址。
2. 放行方向为**易宝 → 商户**（入站），与商户调用 API 的出站方向不同。
3. 配置后可用沙箱联调或 `scripts/rsa/mock_notify.py` 验证回调链路（见 `结果通知机制说明.md`）。

## 四、排障关联

回调收不到时，除验签与 `notifyUrl` 配置外，请核对：

1. 防火墙/安全组是否拦截了上表 IP 或 `out.yeepay.com` 解析结果。
2. 负载均衡、WAF、CDN 是否丢弃了 POST 请求体。
3. 详见 `../../troubleshooting.md` 第二节「回调问题排查」。
