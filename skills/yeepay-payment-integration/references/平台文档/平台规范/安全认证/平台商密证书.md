# 平台商密证书

## 一、背景

自 2021 年 3 月起，启用**商密算法**的商户（RSA 不受此影响），平台使用 CFCA 颁发的商密平台证书标识易宝开放平台身份。证书设有有效期，商户须在过期前完成轮换与缓存更新。

本文说明轮换机制、查询接口（V2）及商户侧实现建议。本地联调可用 `scripts/sm/list_platform_certs.py`。

## 二、证书轮换与平滑切换

易宝开放平台**同时兼容**新旧平台证书：

- 旧证书**过期前 5 天至过期当天**，新证书开始逐步用于 API 应答签名与结果通知。
- 商户须根据 HTTP 头 **`x-yop-sign-serial-no`**（证书序列号）选用对应版本的平台公钥验签。
- 新证书启用前约 **24 小时**会出现在证书列表中，便于商户提前加载。

使用易宝官方 SDK 时，SDK 已封装证书惰性加载与切换，可忽略手动轮换细节。

## 三、商户侧最佳实践

旧证书过期前 5 天内，平台可能**同时使用**新旧证书，商户系统须支持**多本**平台证书并存。

### 使用易宝 SDK（推荐）

默认惰性加载：当本地无某序列号对应证书时，自动调用下文 V2 接口拉取并缓存（磁盘 + 内存）。

### 商户自实现

1. **多级缓存**：内存 + 磁盘缓存平台证书，降低延迟并抵御网络抖动。
2. **按序列号验签**：根据 `x-yop-sign-serial-no` 选取公钥；缓存未命中时**立即**调用证书列表接口拉取后再验签。
3. **定期刷新**：刷新间隔 **小于 12 小时**（平台建议）；**不要**硬编码单一平台证书。
4. **作废过期证书**：到达 `expireDate` 的证书应从商户侧缓存中删除；平台不会用过期证书签名。

## 四、获取证书列表（V2）

### 调用说明

- 接口：`GET /rest/v2.0/yop/platform/certs`
- 须使用 **SM2/SM3 鉴权**（`YOP-SM2-SM3`），商户侧为已配置的 **CFCA 商密私钥**。
- 频率限制：单商户约 **1000 次/s**。

### 请求参数

Content-Type：`application/x-www-form-urlencoded`（GET 查询串）

| 参数 | 类型 | 必填 | 描述 | 示例 |
| --- | --- | --- | --- | --- |
| certType | string | 否 | 证书类型 | `SM2` |
| serialNo | string | 否 | 指定序列号，不填则返回全部可用 | `4632476314` |

### 响应参数

Content-Type：`application/json`

| 参数 | 类型 | 描述 |
| --- | --- | --- |
| data | object[] | 证书列表 |
| data[].serialNo | string | 证书序列号（与 `x-yop-sign-serial-no` 对应，多为十进制字符串） |
| data[].effectiveDate | string | 生效时间，如 `2023-02-20 14:53:30` |
| data[].expireDate | string | 失效时间 |
| data[].cert | string | **PEM 格式**完整证书文本 |

### 响应示例

```json
{
  "data": [
    {
      "serialNo": "289798445125",
      "effectiveDate": "2021-03-03 15:11:10",
      "expireDate": "2023-03-03 15:11:10",
      "cert": "-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----"
    }
  ]
}
```

实际 HTTP 响应可能外包一层 `state` / `result`，以在线文档与网关返回为准；`list_platform_certs.py` 已做常见结构兼容。

### V1（已废弃）

`GET /rest/v1.0/yop/platform/certs` 返回加密的 `encryptCertificate` 对象，需商户加密密钥解密。**请使用 V2**，直接返回 PEM `cert` 字段。

## 五、本地联调

在 `scripts/` 目录下执行：

```bash
python sm/list_platform_certs.py \
  --appkey 你的appKey \
  --key ./keys/sm2_private_pkcs8.pem
```

沙箱示例：

```bash
python sm/list_platform_certs.py \
  --appkey sandbox_sm_xxx \
  --key ./keys/sm2_private_pkcs8.pem \
  --base-url https://sandbox.yeepay.com/yop-center \
  --save-dir ./certs/yop
```

## 六、证书使用与验签

1. 将响应中的 `cert` 保存为 PEM 后，可用 OpenSSL 导出公钥：

```bash
openssl x509 -in yop_platform.cer -pubkey -noout > yop_platform_public.pem
```

2. 列表中**未过期**且 `effectiveDate` 已到的证书均可并存；验签时以报文头 `x-yop-sign-serial-no` 为准。
3. SM2 应答验签、结果通知验签均依赖正确的平台公钥，协议见 `结果通知(SM2).md`、`报文加密机制(SM).md`、`请求签名协议.md` §应答验签；本地工具 `scripts/common/response_verify.py`、`scripts/tools/verify_response.py`。
