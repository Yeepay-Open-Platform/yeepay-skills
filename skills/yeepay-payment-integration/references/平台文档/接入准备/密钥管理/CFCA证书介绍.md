# CFCA证书介绍

## 一、什么是 CFCA 证书

数字证书是一段包含用户公开密钥、用户信息、颁发机构信息、证书序列号、有效时间、发证机关（CA 中心）名称及 CA 数字签名的数据，格式遵循 X.509 V3 国际标准。由 CFCA 颁发的数字证书简称 CFCA 证书。

## 二、是否需要申请 CFCA 证书

- **商密（SM）对接**：必须申请并激活 CFCA 证书。
- **RSA 对接**：一般收单/查询类业务可用普通 RSA 密钥；**资金出款类接口必须使用 CFCA 证书**（见下节）。

### 资金出款类接口必须使用 CFCA 证书（强约束）

**`account`（账户）与 `balance`（代付代发）分组**下的资金出款类接口，平台强制要求使用 CFCA 证书对应的私钥签名：

| 分组 | 路径特征 | 典型接口 |
| --- | --- | --- |
| `account`（账户） | `/rest/v1.0/account/...` | 提现下单 `/rest/v1.0/account/withdraw/order` |
| `balance`（代付代发） | `/rest/v1.0/balance/...` | 单笔代付 `/rest/v1.0/balance/transfer_send`、批量代付 `/rest/v1.0/balance/transfer_batch_send` |

- 仅配置普通 RSA 文本密钥（非证书格式）时，调用此类接口会被平台**拒绝并报错**。
- 结算接口（`/rest/v1.0/settle/...`，`settle` 分组）不在上述强制范围内。
- 接入出款产品前，须先完成证书申请与激活（`CFCA证书制备.md`），并在控制台密钥配置中将**密钥格式选「证书」**（`密钥配置(RSA).md` / `密钥配置(SM).md`）。
- 此类接口通常还要求配置 IP 白名单，见 `开始对接/配置IP白名单.md`。

## 三、平台支持的 CFCA 证书类型

- **CFCA 商密证书（SM2）**
- **CFCA 国际证书（RSA2048）**（高风险 RSA 场景）

## 四、CFCA 证书在商户与易宝交互中的作用

1. **商户 → 易宝（请求）**：商户使用 CFCA 证书对应的私钥对请求签名；易宝用商户上传的证书公钥验签。
2. **易宝 → 商户（结果通知）**：易宝使用平台私钥签名/加密；商户使用易宝公钥验签，使用商户私钥解密（SM2 场景需按 `结果通知(SM2).md` 回写响应签名）。

**商密平台证书**与商户 CFCA 证书不同：平台证书有有效期且会轮换，商户须按 `x-yop-sign-serial-no` 选用公钥，并定期拉取证书列表，见 `平台规范/安全认证/平台商密证书.md`。

申请与激活步骤见 `CFCA证书制备.md`；控制台配置见 `密钥配置(SM).md` 或 `密钥配置(RSA).md`（证书格式）。
