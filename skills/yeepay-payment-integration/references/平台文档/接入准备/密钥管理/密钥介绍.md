# 密钥介绍

## 一、平台支持的密钥类型

### 1. 商密密钥（SM）

国家商用密码管理办公室制定了一系列密码标准，包括 SM1、SM2、SM3、SM4、SM7、SM9、祖冲之密码算法（ZUC）等。其中 SM1、SM4、SM7、祖冲之（ZUC）为对称算法；**SM2、SM9** 为非对称算法；**SM3** 为哈希算法。平台商密场景使用 **SM2 + CFCA 证书**。

**制备前准备**：需申请、激活并配置商密 CFCA 证书（见 `CFCA证书介绍.md`、`CFCA证书制备.md`）。

### 2. 国际密钥（RSA）

国际密钥（RSA）是目前最有影响力的非对称加密算法，能够抵抗绝大多数已知密码攻击。接口采用 RSA 签名加密，适用于多语言接入场景。**制备前准备**：仅需配置国际密钥（RSA），无需 CFCA（一般业务）；**`account`（账户）、`balance`（代付代发）分组下的资金出款类接口（如提现、代付代发）必须使用 CFCA 证书**，见 `CFCA证书介绍.md`。

## 二、需要制备哪些密钥

| 类型 | 商户侧需配置 | 平台侧 |
| --- | --- | --- |
| RSA | 商户 RSA 公钥（私钥本地保管） | 易宝 RSA 公钥（控制台可复制） |
| SM | 商密 CFCA 证书（SM2） | 易宝商密证书（须定期拉取，见 `平台规范/安全认证/平台商密证书.md`） |

## 四、制备商户密钥

密钥包含一对公钥和私钥：私钥由开发者保存用于签名/解密；公钥上传平台用于验签/加密。`account`（账户）、`balance`（代付代发）分组下的资金出款类接口（如提现 `/rest/v1.0/account/withdraw/order`、代付 `/rest/v1.0/balance/transfer_send`）**必须使用 CFCA 证书**，仅配置普通文本密钥调用会被平台拒绝，见 `CFCA证书介绍.md`。

### 方式一：密钥工具（推荐）

操作步骤见 `工具与支持/开发工具/密钥工具.md` §生成密钥。

### 方式二：OpenSSL 生成 RSA（非 Java 或脚本场景）

```bash
openssl genrsa -out app_private_key.pem 2048
openssl pkcs8 -topk8 -inform PEM -in app_private_key.pem -outform PEM -nocrypt -out app_private_key_pkcs8.pem  # Java 需要
openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem
```

### 注意事项

- 每次生成会得到不同密钥对；平台不保存商户私钥，务必妥善保管。
- 切换开发语言时若密钥格式不同需重新生成。
- 商户私钥须与上传的公钥匹配；控制台验证公钥见 `密钥配置(RSA).md` / `密钥配置(SM).md`。
